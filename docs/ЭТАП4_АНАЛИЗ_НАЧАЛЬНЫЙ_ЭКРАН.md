# Этап 4. Глубокий анализ: начальный экран

Дата анализа: 2026-07-18  
Статус: **реализовано** — стабильные ID, мультивалютность, отчёт взаиморасчётов, подсказки периодов.

---

## 1. Роль обработки

`Обработка.НачальныйЭкран` — главный экран МП:
- сводка расходов/доходов за сегодня / неделю / месяц (по валюте учёта);
- список счетов с остатками;
- быстрый ввод прихода/расхода;
- переход к отчётам;
- (побочно) медиаплеер — вне объёма этапа 4, относится к этапу 6.

Модуль: `DataProcessors/НачальныйЭкран/Forms/Форма/Module.bsl`.

---

## 2. Как строится UI

### 2.1. Сводка доходов/расходов

`СформироватьДанныеПоРасходамДоходам` → запрос к `РегистрНакопления.{Доходы|Расходы}.Обороты` → динамические декорации в группы `ГруппаДанные*Лево/Право`.

Имена элементов стабильны: `ДекорацияРасходыСегодня`, `ДекорацияРасходыСегодняСумма` и т.п. При повторном вызове `ДобавитьДекорациюНаФорму` удаляет и создаёт заново — для сводки это приемлемо.

### 2.2. Список счетов

`СформироватьСписокСчетов`:

```bsl
НаименованиеСчета = СтрЗаменить(Выборка.Счет, " ", "");
ДобавитьГруппуНаФорму(..., "Группа" + НаименованиеСчета, ...);
ДобавитьКартинкуСчетаДекорации(..., "ДекорацияКартинка" + НаименованиеСчета, ...);
ДобавитьДекорациюНаФорму(..., "Декорация" + НаименованиеСчета, Выборка.Счет, ...);
ДобавитьДекорациюНаФорму(..., "Декорация" + НаименованиеСчета + "Сумма", Строка(Выборка.Сумма), ...);
```

Запрос:

```
Справочник.Счета
  ЛЕВОЕ СОЕДИНЕНИЕ РН.Баланс.Остатки
  ПО Баланс.Счета = Счет.Ссылка
```

Без свёртки по валюте: одна строка выборки = пара **счёт + валюта** (или счёт без остатка).

## 3. Проблемы (приоритет)

### P0 / Важно — имена элементов из наименования (п. 4.1)

| Риск | Пример |
|------|--------|
| Коллизия имён | «Банк 1» и «Банк1» → один ключ `Банк1` |
| Переименование | меняется имя элемента → ломается логика, возможны «хвосты» старых групп |
| Навигация по сумме | для декорации суммы `Подсказка = "Команда" + Заголовок`, а заголовок — **число**, не имя счёта |

Сейчас клик по счёту:

```bsl
ИмяСчета = СтрЗаменить(Элемент.Подсказка, "Команда", "");
Счет = НайтиСчетПоНаименованию(ИмяСчета);
```

Поиск по наименованию хрупкий (не уникален, зависит от локали/пробелов).

**Целевое:** технический ключ элемента = стабильный ID (код / UUID ссылки); в `Подсказка` или отдельном реквизите формы — ссылка на счёт (и при необходимости валюта).

### P0 / Важно — несколько валют на одном счёте (п. 4.2)

При двух валютах на одном счёте цикл дважды создаёт элементы с **одинаковым** `НаименованиеСчета`:
1. первая валюта — создаёт группу;
2. вторая — находит группу, **удаляет**, создаёт заново только со второй суммой.

На экране остаётся **одна** валюта (последняя в выборке). Остальные теряются визуально.

**Варианты решения:**
- **A (рекомендуется):** строка на каждую пару счёт+валюта; уникальный ключ `Счет_<UID>_<UIDВалюты>`; в заголовке «Кошелёк · KZT»;
- **B:** одна строка на счёт, суммы валют через запятую / вложенный список;
- **C:** показывать только валюту учёта (фильтр в запросе) — проще, но хуже для мультивалютности.

### Важно — пустая команда «Отчет по взаиморасчётам» (п. 4.3)

```bsl
Процедура СформироватьОтчетПоВзаиморасчетам(Команда)
	// Вставить содержимое обработчика.
КонецПроцедуры
```

Кнопка на странице отчётов **есть**, обработчик пустой.  
При этом клик по счёту уже открывает `Отчет.ОтчетПоВзаиморасчетам.Форма.ФормаОтчета` с параметром `Счет`.

**Рекомендация:** реализовать команду — открыть тот же отчёт без обязательного отбора по счёту (или с пустым счётом / всеми счетами), по аналогии с отчётами доходов/расходов.

### Важно (смежное) — подсказки периодов сводки

В `ДобавитьДекорациюНаФорму` подсказка для сводки ставится так:

```bsl
Если СтрНайти(ИмяРодитель, "Расходы") ИЛИ СтрНайти(ИмяРодитель, "Доходы") Тогда
```

Родители сейчас: `ГруппаДанныеРасходовЛево`, `ГруппаДанныеДоходовЛево`.  
Подстроки `"Расходы"` / `"Доходы"` в `"Расходов"` / `"Доходов"` **не находятся** → ветка не срабатывает → `Подсказка` для периодов часто пустая → `ОбработкаНавигационнойСсылки` открывает отчёт некорректно.

В этап 4 стоит включить: явный формат подсказки, например `Расходы|Сегодня`, `Доходы|Месяц`.

### Желательно — очистка перед пересборкой

`ПослеЗакрытияФормыДокумента` снова вызывает `ПодготовитьФормуНаСервере`.  
Сводка пересоздаётся по фиксированным именам. Счета — тоже, но удалённый из справочника счёт может оставить старую группу, если имени больше нет в выборке.

**Целевое:** перед `СформироватьСписокСчетов` очищать подчинённые элементы `ГруппаСчета`.

### Вне этапа 4

Медиаплеер / рингтон — этап 6.  
`ДобавитьГруппуНаФорму` с `Попытка` + `Сообщить` — этап 6 / рефакторинг UI-хелперов.

## 4. Рекомендуемый объём правок этапа 4

1. **Ключи элементов счетов** — по ссылке (и валюте), не по наименованию.
2. **Вывод мультивалютных остатков** — вариант A (строка на счёт+валюту).
3. **Навигация** — `Подсказка` вида `Счет|<уникальныйИдентификатор>` (+ валюта при необходимости); поиск по ссылке, не по наименованию.
4. **Команда отчёта взаиморасчётов** — открыть `ОтчетПоВзаиморасчетам`.
5. **Подсказки периодов** — явный контракт `Вид|Период`.
6. **Очистка** `ГруппаСчета` перед пересборкой списка.

## 6. Критерий готовности

- Два счёта с похожими именами не конфликтуют.
- Счёт с остатками в двух валютах показывает обе суммы.
- Кнопка «Отчет по взаиморасчётам» открывает отчёт.
- Клик по периоду сводки открывает соответствующий отчёт за нужный период.
- После записи операции список счетов обновляется без «хвостов».
