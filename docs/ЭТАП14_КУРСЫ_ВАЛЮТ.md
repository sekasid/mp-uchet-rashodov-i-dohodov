# Этап 14. Регламентное обновление курсов валют

Цель: ежедневное обновление курсов RUB / USD / EUR относительно валюты учёта, если не используется режим «последний известный курс».

Источник в Notion: [Этап 14 — Регламентное обновление курсов валют](https://app.notion.com/p/0d421f9095e64535ad29bbd91668f2fa).

## Что сделано

| Пункт | Статус |
|-------|--------|
| Константа `ИспользоватьПоследнийИзвестныйКурс` влияет на чтение курсов | готово (было) |
| Константа `АвтообновлениеКурсовВалют` | готово |
| UI: автообновление доступно только при выкл. «последний известный курс» | готово |
| РЗ `ОбновлениеКурсовВалют` (ежедневно в 06:00) | готово |
| Загрузка RUB / USD / EUR относительно `ВалютаУчета` | готово |
| Источник: `open.er-api.com` (HTTPS) | готово |
| Не перезаписывать курс пустым / ошибочным значением | готово |
| Журнал регистрации «Обновление курсов валют» | готово |
| Команда «Обновить курсы сейчас» на форме констант | готово |
| Автозагрузка при старте приложения (если нет курсов на сегодня) | готово |
| Документы используют курсы по выбранному режиму | готово (было) |

## Как пользоваться

1. **Настройки → Константы**
2. Выключить «Использовать последний известный курс», если нужны точные курсы на дату.
3. Включить «Ежедневное автообновление курсов (06:00)» при необходимости.
4. Нажать **«Обновить курсы сейчас»** — сообщение: успешно / нет изменений / ошибка.

## Технические детали

- Модуль: `ОбщийМодульСервер_КурсыВалют`
- API: `GET https://open.er-api.com/v6/latest/{КодВалютыУчёта}`
- Курс в регистре: единиц валюты учёта за 1 единицу валюты операции (`1 / rate_api`)
- На мобильном клиенте регламентные задания могут быть недоступны — тогда срабатывает обновление при старте + ручная команда
- В `usedMobileApplicationFunctionalities` включён `LocalNetwork` для HTTPS

- [ ] Документ в иностранной валюте проводится при наличии курса
