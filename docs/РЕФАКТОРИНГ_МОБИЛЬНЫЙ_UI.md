# Рефакторинг мобильного UI

Дата: 2026-07-18  
Платформа: 1С МП 8.3.22 (управляемые формы)

---

## Ограничения

На мобильной 1С нет свободного UI-kit (карточки, тени, нижний tab bar, произвольные анимации).  
Доступны: группы, порядок полей, картинки, цвета, шрифты, кнопки, динамические декорации.

Поэтому «красота» = **иерархия, читаемость, явные действия, спокойная палитра**, а не pixel-perfect как в нативном приложении.

---

## Что сейчас мешает

| Проблема | Где |
|----------|-----|
| Кислотные цвета (`Yellow`, `Chartreuse`) | Начальный экран |
| Добавление операции — мелкая иконка без текста | Шапка расходов/доходов |
| Отчёты — клик по картинке | Начальный экран |
| Сводка — россыпь надписей, не «карточки» | Динамические декорации |
| Шрифт 10 pt на всё подряд | `ОбщийМодульКлиент` |
| Быстрый ввод собирается только кодом | `ОбщаяФормаДокумента` |
| Списки документов — «толстая» таблица | Формы списков |

---

## Направления

### A. Спокойный mobile-first (минимум риска)
Нейтральные цвета, текстовые кнопки, порядок полей. Быстро, надёжно, визуально скромно.

### B. Карточный финансовый экран ★ рекомендуется
1. Сверху сводка периодов  
2. Две крупные кнопки **Расход** / **Доход**  
3. Блок счетов (иконка + название + остаток)  
4. Отчёты — обычные кнопки  

Плюс: заметно удобнее на телефоне.  
Минус: переработка `Form.form` начального экрана и частично быстрого ввода.

### C. Журнал операций как главный экран
Фокус на последних операциях; сводка и счета — отдельные экраны. Меняет IA приложения.

---

## Реализация главной: HTML-гибрид (этап 7)

Главная страница — HTML (`ПолеHTML`), быстрый ввод и отчёты — существующие формы 1С.  
Подробности: [`ЭТАП7_HTML_ГЛАВНАЯ.md`](ЭТАП7_HTML_ГЛАВНАЯ.md).

Дальше по желанию: доработать вёрстку макета, добавить «Перевод»/«Долги», HTML для других экранов.

### Осталось вне HTML-главной

1. В быстром вводе: **сумма первой**, затем счёт/категория  
2. Не красить кнопку зелёным овалом из helper по умолчанию  
3. Списки документов: компактнее  
4. Календарь: «Готово» вместо «Закрыть»

## Палитра (предложение)

| Роль | Цвет | Назначение |
|------|------|------------|
| Фон | без заливки / мягкий серый | секции |
| Расход | текст/иконка, без кислотного фона | смысл |
| Доход | текст/иконка | смысл |
| Основное действие | спокойный зелёный или системный акцент | кнопка записи |
| Вторичное | без заливки | отчёты, назад |

Цвет не единственный носитель смысла: всегда подпись «Расход»/«Доход».
