# Этап 2. Анализ и решение: первый запуск

## Было

1. В `ПередНачаломРаботыСистемы` при `ПервыйЗапуск = Ложь` открывалась форма констант и **сразу** ставился флаг в `Истина`.
2. Виды предопределённых счетов заполнялись только при `НЕ ПервыйЗапуск`, то есть фактически **после** установки флага уже не выполнялись.
3. `ПриЗакрытии` формы вызывал `ЗавершитьРаботуСистемы` — приложение завершалось после настроек.
4. Имя константы «Первый запуск» при значении `Истина` означало «настройка уже пройдена» — семантика перевёрнута.

## Стало

1. Флаг ставится только в `ПослеЗаписиНаСервере` формы констант, если валюта заполнена.
2. Закрытие формы без валюты блокируется в `ПередЗакрытием`.
3. `ЗавершитьРаботуСистемы` убран.
4. Виды счетов инициализируются при каждом старте идемпотентно (запись только при отличии).
5. Синоним константы: «Начальная настройка выполнена»; API: `НачальнаяНастройкаВыполнена` / `УстановитьНачальнаяНастройкаВыполнена`. Имя метаданных `ПервыйЗапуск` сохранено (совместимость данных).
