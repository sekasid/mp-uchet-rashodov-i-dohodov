# Этап 10. Контрольный тестовый чек-лист

Цель: минимальный набор сценариев для быстрой проверки после каждой доработки.

Источник в Notion: [Этап 10 — Контрольный тестовый чек-лист](https://app.notion.com/p/8c5e034dd1c04bc5a3346d084ed5c732).

## Готовность кода (аудит)

| Сценарий | Статус в коде | Где |
|----------|---------------|-----|
| Валюта учёта обязательна на первом запуске | готово | `ManagedApplicationModule`, `ФормаКонстант` |
| Счета: Наличные / Банковская карта / Банковский счёт | готово | предопределённые `Счета.Кошелек` / `Карта` / `БанковскийСчет` + синхронизация наименований при старте |
| Категории доходов: Зарплата, Прочий доход | готово | предопределённые в `КатегорииДоходов` |
| Категории расходов: Продукты, Транспорт, Коммунальные услуги | готово | предопределённые в `КатегорииРасходов` |
| Расход без счёта → ошибка | готово | `Документ.Расход` / `fillChecking` + `ПроверитьРеквизитыДокумента` |
| Сумма 0 → ошибка | готово | `ОбработкаПроверкиЗаполнения` Доходы/Расход |
| Остаток по счёту после дохода/расхода | готово | РН `Баланс`, движения при проведении |
| Отчёты за сегодня | готово | чип «Сегодня» в HTML-отчётах |

Ручной прогон на устройстве / в клиенте обязателен: код закрывает проверки, но критерии «все тесты проходят» подтверждаются только прогоном.

## Тестовые данные

После обновления конфигурации БД:

1. Валюта учёта: **KZT** (выбирается на первом запуске).
2. Счета (появятся автоматически):
   - Наличные (`Счета.Кошелек`);
   - Банковская карта (`Счета.Карта`);
   - Банковский счёт (`Счета.БанковскийСчет`).
3. Категории доходов:
   - Зарплата;
   - Прочий доход.
4. Категории расходов:
   - Продукты;
   - Транспорт;
   - Коммунальные услуги.

## Минимальные тесты (ручной прогон)

- [ ] Первый запуск: без валюты форму настроек закрыть нельзя.
- [ ] Доход **100 000 KZT** на **Банковскую карту**, категория **Зарплата**.
- [ ] Расход **5 000 KZT** с **Банковской карты**, категория **Продукты**.
- [ ] Остаток по банковской карте: **95 000 KZT** (на главной и в списке счетов).
- [ ] Отчёт по доходам → период **Сегодня** → сумма 100 000.
- [ ] Отчёт по расходам → период **Сегодня** → сумма 5 000.
- [ ] Отчёт по взаиморасчётам → период **Сегодня** → доходы 100 000, расходы 5 000, остаток +95 000.
- [ ] Расход без счёта — запись/проведение отклонены с ошибкой.
- [ ] Расход с суммой **0** — ошибка.
- [ ] Документ с пустыми обязательными реквизитами — ошибка заполнения.

## Критерии готовности этапа

- [x] Тестовые справочники и валидации подготовлены в конфигурации.
- [x] Чек-лист зафиксирован в репозитории (`docs/ЭТАП10_ТЕСТОВЫЙ_ЧЕКЛИСТ.md`).
- [ ] Все пункты ручного прогона отмечены на устройстве.
- [ ] Ошибки показываются пользователю (проверить визуально при негативных тестах).
- [ ] Остатки и отчёты совпадают с движениями документов.

## Что сделано в конфигурации для этапа 10

1. Добавлены предопределённые категории доходов и расходов по чек-листу.
2. Наименования предопределённых счетов приведены к чек-листу (Наличные / Банковская карта / Банковский счёт).
3. При старте синхронизируются вид счёта и наименование предопределённых счетов.
